# 项目管理系统需求文档

## 1. 系统概述

### 1.1 系统名称
项目管理系统

### 1.2 系统描述
一个用于管理45天学习计划的项目管理系统，帮助用户跟踪学习进度、完成任务打卡、管理学习内容。系统包含多个模块，每个模块都支持增删查改等基础操作。\n
## 2. 核心功能模块

### 2.1 仪表盘首页
- 今日任务模块：展示当日待完成任务
- 支持编辑每日任务
- 快速查看整体学习进度
- 时间轴功能：可视化展示过去、现在、未来的每日任务，支持按时间线查看任务分布与进展

### 2.2 学习任务管理
- Linux环境下的编程学习计划管理
- 基于PyTorch的医学影像分析项目实战管理（包含Kaggle、Grand Chanllege比赛练习记录）\n- 医学影像相关基础论文阅读管理（每天一篇文献阅读记录）\n- CS336大模型课程学习进度管理
- 每日代码练习打卡（每天围绕一个具体小目标，写代码30分钟）\n- Docker、Git、计算机网络学习进度管理
- 项目管理系统开发进度管理

### 2.3 任务增删查改
- 添加新学习任务或子任务
- 删除已完成或不需要的任务
- 查询所有任务列表或按分类查询
- 修改任务状态、内容或优先级

### 2.4 每日任务管理
- 每日任务标题：为每日任务设置独立标题
- 任务信息：记录每日任务的具体内容与描述
- 优先级：设置任务优先级（高、中、低）\n- 状态：标记任务状态（待开始、进行中、已完成）\n- 关联母任务：将每日任务关联至对应的母任务模块

### 2.5 周总结功能
- 本周目标：记录本周设定的学习目标
- 本周成就：总结本周完成的成就与进展
- 本周挑战：记录本周遇到的困难与挑战
- 下周目标：规划下周的学习目标
- 支持对以上四个部分的增删查改操作

### 2.6 进度跟踪
- 每日学习打卡记录
- 45天学习计划进度统计
- 各模块完成情况可视化展示

### 2.7 数据管理
- 任务数据存储与管理
- 学习记录持久化保存
- 支持数据导出功能

## 3. 使用场景
用户通过该系统管理45天内的多个学习任务，包括技术学习、项目实战、论文阅读、课程学习等内容。系统提供完善的增删查改功能，帮助用户灵活调整学习计划，并通过打卡机制记录每日学习进度。仪表盘首页展示今日任务，支持快速编辑；周总结功能帮助用户回顾与规划学习进度。时间轴功能可视化展示每日任务的时间分布，便于用户整体把握学习进度。
