# 项目管理系统需求文档

## 1. 系统概述

### 1.1 系统名称
项目管理系统

### 1.2 系统描述
一个用于管理45天学习计划的项目管理系统，帮助用户跟踪学习进度、完成任务打卡、管理学习内容。系统包含多个模块，每个模块都支持增删查改等基础操作。\n
## 2. 核心功能模块

### 2.1 学习任务管理
- Linux环境下的编程学习计划管理
- 基于PyTorch的医学影像分析项目实战管理（包含Kaggle、Grand Chanllege比赛练习记录）\n- 医学影像相关基础论文阅读管理（每天一篇文献阅读记录）\n- CS336大模型课程学习进度管理
- 每日代码练习打卡（每天围绕一个具体小目标，写代码30分钟）\n- Docker、Git、计算机网络学习进度管理
- 项目管理系统开发进度管理

### 2.2 任务增删查改
- 添加新学习任务或子任务
- 删除已完成或不需要的任务
- 查询所有任务列表或按分类查询
- 修改任务状态、内容或优先级

### 2.3 进度跟踪
- 每日学习打卡记录
- 45天学习计划进度统计
- 各模块完成情况可视化展示

### 2.4 数据管理
- 任务数据存储与管理
- 学习记录持久化保存
- 支持数据导出功能

## 3. 使用场景
用户通过该系统管理45天内的多个学习任务，包括技术学习、项目实战、论文阅读、课程学习等内容。系统提供完善的增删查改功能，帮助用户灵活调整学习计划，并通过打卡机制记录每日学习进度。
